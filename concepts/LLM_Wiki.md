# LLM Wiki 知识库构建模式

## 概念来源

- **提出者**: Andrej Karpathy

- **时间**: 2026年（具体日期不详）

## 核心思想

### 传统 RAG 的问题

- 每次查询重新检索相关片段
- LLM 每次都要从零开始推理
- 没有知识积累
- 复杂问题需要综合多个文档时效率低

### LLM Wiki 的解决方案

- **持久化知识结构**: LLM 增量构建和维护 wiki
- **结构化 markdown 文件**: 介于原始文档和人类查询之间
- **持续积累**: 添加新源时，LLM 提取关键信息并整合到现有 wiki
- **交叉引用**: 页面间相互链接，形成知识图谱
- **矛盾标记**: 新数据与原声明冲突时自动标记

## 关键差异

| 特性 | 传统 RAG | LLM Wiki |
|------|----------|----------|
| 知识存储 | 原始文档 + 向量索引 | 结构化 markdown wiki |
| 查询时处理 | 检索 + 重新推理 | 直接读取已编译的知识 |
| 知识积累 | 无（每次重新推导） | 有（持续更新） |
| 交叉引用 | 无 | 有（自动维护） |
| 矛盾处理 | 无 | 自动标记 |

## 工作流程

1. **人类提供信息源**: 文章、论文、书籍、笔记
2. **LLM 处理**: 读取源，提取关键信息
3. **LLM 更新 wiki**: 更新实体页面、修订主题摘要、标记矛盾
4. **人类浏览**: 用 Obsidian 等工具查看 wiki，跟随链接
5. **循环**: 每次对话后，LLM 更新相关 wiki 页面

## 应用场景

### 个人

- 追踪目标、健康、心理、自我提升
- 归档日记条目、文章、播客笔记
- 构建随时间的自我结构化图片

### 研究

- 数周或数月深入研究一个主题
- 阅读论文、文章、报告
- 增量构建综合 wiki 和演化论文

### 阅读书籍

- 逐章归档
- 构建角色、主题、情节线索页面
- 查看它们如何连接

## 工具链

- **LLM Agent**: OpenAI Codex, Claude Code, OpenCode / Pi 等
- **IDE**: Obsidian（查看 wiki、图谱视图、跟随链接）
- **存储**: 本地 markdown 文件 + Git 版本控制

## 优势

1. **知识复用**: 一次编译，多次查询
2. **深度推理**: wiki 已综合多源信息
3. **可追溯**: Git 历史记录知识演化
4. **人机协作**: 人类负责探索和问题，LLM 负责总结和归档

## 实施建议

1. **从简单开始**: 先构建一个小 wiki（如个人知识库）
2. **选择合适工具**: Obsidian + LLM Agent
3. **建立流程**: 定期（如每周）让 LLM 更新 wiki
4. **人工审核**: 定期浏览 wiki，确保质量
